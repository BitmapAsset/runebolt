# RuneBolt User Journey Maps

*Detailed journey maps for key user personas*

## Journey A: First-Time Consumer User

**Persona:** Sarah, 28, discovers DOG at a dinner with friends
**Goal:** Send DOG to split a dinner bill
**Tech Comfort:** Uses Venmo, Cash App, but new to crypto

---

### Stage 1: Discovery

**Context:** At dinner, friend Mike pays with DOG via RuneBolt. Sarah is intrigued.

**Touchpoints:**
- Sees Mike tap phone and instantly send DOG
- Hears: "It's like Venmo but for DOG. Just use RuneBolt."
- Mike shares: "My handle is @mike — just search for me"

**Emotions:** Curious → Intrigued → Slightly skeptical

**Pain Points:** None yet — positive social proof

**Opportunities:** 
- Word-of-mouth is powerful
- Social context reduces anxiety

---

### Stage 2: Onboarding (Target: 90 seconds)

**Touchpoint 2.1: Landing Page**
```
Sarah opens runebolt.io on her phone
→ Sees clean, minimal design
→ Headline: "Send DOG instantly, anywhere"
→ Clear CTA: "Get Started - Free"
→ No overwhelming crypto jargon visible
```
**Emotion:** Curious → Reassured by simplicity

**Touchpoint 2.2: Account Creation**
```
Taps "Get Started"
→ Sees: "Continue with Google / Apple / Email"
→ Chooses Google (familiar, fast)
→ Account created in 3 seconds
```
**Emotion:** Relieved → This is easy

**Touchpoint 2.3: Username Selection**
```
Prompted to choose username
→ Types "sarah" → "Available!" 
→ Tooltip: "This is how friends send you DOG"
→ Picks @sarah
```
**Emotion:** Excited → This feels like Venmo

**Touchpoint 2.4: First Deposit**
```
Sees options:
→ [Buy DOG with Card] — "Hmm, not ready for that"
→ Or "Send DOG to: runebolt.io/sarah"
→ Copies address, switches to her wallet app
→ Sends 10,000 DOG (~$50)
→ Sees: "Your deposit is on the way! ~10 minutes"
```
**Emotion:** Confident → I know how this works

**Touchpoint 2.5: Success**
```
Sees: "🎉 You're all set!"
→ [Send DOG to a friend] [Explore]
→ Taps "Send to friend"
```
**Emotion:** Ready → Let's do this

**Total Time:** ~90 seconds
**Steps:** 5 (but feels like 3)

---

### Stage 3: First Send

**Touchpoint 3.1: Send Screen**
```
Search bar: "Who are you sending to?"
→ Types "mike"
→ Sees @mike with profile picture
→ Taps to select
```
**Emotion:** Familiar → Just like Venmo

**Touchpoint 3.2: Amount Entry**
```
Amount input with numeric keypad
→ Enters 2,500 DOG
→ Sees "$12.50" equivalent below
→ Memo field: "🍕 Dinner split"
```
**Emotion:** Confident → Clear what I'm sending

**Touchpoint 3.3: Confirmation**
```
Review screen:
→ "Send 2,500 DOG ($12.50) to @mike?"
→ Shows memo, confirms details
→ [Send] button prominent
→ Taps Send
```
**Emotion:** Certain → Double-check feels safe

**Touchpoint 3.4: Instant Confirmation**
```
Animation: Sending → Sent!
→ "Mike received 2,500 DOG!"
→ Confetti animation
→ "🍕 Dinner split" shown
→ Back to home with updated balance
```
**Emotion:** Delighted → That was FAST!

**Total Time:** ~30 seconds

---

### Stage 4: Regular Use

**Week 1-2: Exploration**
- Opens app to check balance
- Sends DOG to 2-3 more friends
- Receives DOG from Mike
- Discovers transaction history

**Week 3-4: Habits Form**
- Uses RuneBolt as default for DOG
- Has 5-10 regular contacts
- Understands "instant" means no waiting
- Shares runebolt.io/sarah on social

**Touchpoints:**
- Push notification: "You received 1,000 DOG from @alice!"
- Home screen shows recent contacts
- Quick-send from notification
- Balance always visible but not overwhelming

**Emotion:** Comfortable → This is my default

---

### Stage 5: Advocacy

**Month 2+:**
- Invites friends via referral code
- Shares RuneBolt handle on Twitter bio
- Suggests merchants accept RuneBolt
- Participates in community Discord

**Why She Advocates:**
- It just works — no explaining channels
- Her friends can use it easily
- She feels "in the know" about DOG
- Social status from early adoption

## Journey B: Merchant Acceptance

**Persona:** Juan, owns coffee shop, tech-curious
**Goal:** Accept DOG payments easily
**Tech Comfort:** Uses Square, comfortable with apps

---

### Stage 1: Awareness

**Context:** Regular customer asks "Do you take DOG?"

**Touchpoints:**
- Customer explains DOG is like Bitcoin but fun
- Customer shows RuneBolt app
- Juan intrigued by instant settlement

**Emotions:** Curious → Open to trying

---

### Stage 2: Setup (Target: 2 minutes)

**Touchpoint 2.1: Business Account**
```
Opens runebolt.io
→ Selects "Business Account"
→ Enters: @juanzcoffee
→ Adds business info
```

**Touchpoint 2.2: Payment Link Generated**
```
Sees: runebolt.io/juanzcoffee
→ QR code generated
→ Options: Print, Save to Photos, Display on tablet
```

**Touchpoint 2.3: First Configuration**
```
Sets:
→ Minimum payment: 500 DOG
→ Auto-convert to USD: Off (for now)
→ Settlement wallet: [his Unisat]
```

---

### Stage 3: First Payment

**Touchpoint 3.1: Customer Pays**
```
Customer scans QR at register
→ Sends 2,500 DOG
→ Juan's tablet shows: "Payment received!"
→ Amount: 2,500 DOG ($12.50)
→ Instant confirmation
```

**Touchpoint 3.2: Receipt**
```
Customer sees: "Paid @juanzcoffee"
→ Optional: Add receipt to email
→ Juan sees transaction in dashboard
```

---

### Stage 4: Daily Operations

**Daily Workflow:**
- QR code displayed at register
- Transactions appear on dashboard
- End of day: review sales
- Weekly: withdraw to main wallet

**Key Features Used:**
- Transaction export for accounting
- Daily settlement summary
- Inventory of DOG holdings
- Customer analytics (repeat vs new)

---

### Stage 5: Scale

**Month 2+:**
- 10-20% of customers pay with DOG
- Regular DOG community visits
- Featured in local "Bitcoin accepted here" lists
- Considering keeping some DOG as investment

## Journey C: Existing Bitcoin User

**Persona:** Alex, Bitcoiner since 2017, frustrated with Lightning
**Goal:** Send DOG easily without channel management
**Tech Comfort:** High — runs own node

---

### Stage 1: Discovery

**Context:** Frustrated with Lightning liquidity issues

**Touchpoints:**
- Tweet about Lightning UX problems
- Reply mentions RuneBolt "just works"
- Skeptical but intrigued

**Emotions:** Frustrated → Skeptical → Curious

---

### Stage 2: Migration

**Touchpoint 2.1: Connect Wallet**
```
Opens RuneBolt
→ Selects "Connect Existing Wallet"
→ Chooses Unisat
→ Signs authentication message
→ Account linked instantly
```

**Touchpoint 2.2: Channel Opening**
```
Prompted to "Activate your account"
→ Sees: "Deposit DOG to enable instant sends"
→ Understands this opens channel
→ Deposits 50,000 DOG
→ Waits for Bitcoin confirmation
```

**Touchpoint 2.3: First Transfer**
```
Sends DOG to friend
→ Instant confirmation
→ No routing issues
→ No liquidity worries
```

**Emotion:** Impressed → Why isn't all crypto this easy?

---

### Stage 3: Parallel Use

**Usage Pattern:**
- Main wallet: Cold storage, large amounts
- RuneBolt: Daily spending, transfers
- Transfers between them as needed

**Appreciates:**
- Speed vs main wallet
- No channel management
- Can always receive (no inbound liquidity issues)

---

### Stage 4: Full Switch

**Month 2+:**
- Sets up auto-forwarding from main wallet
- Gives out RuneBolt address for all DOG receives
- Only touches main wallet for large cold storage
- Recommends to other Bitcoiners

**Why:**
- "It just works" — the highest praise
- No mental overhead
- Bitcoin security with app convenience

## Journey D: Claim Link Recipient

**Persona:** Emma, receives DOG via claim link, no prior crypto
**Goal:** Claim her DOG without setting up complex wallets
**Tech Comfort:** Low — uses apps but no crypto experience

---

### Stage 1: Receiving Link

**Context:** Friend sends link via text: "runebolt.io/claim/abc123"

**Touchpoints:**
- Opens link on phone
- Sees: "🎁 You received 5,000 DOG!"
- Amount shown in USD: "~$25"
- "Claim in 30 seconds"

**Emotion:** Surprised → Curious

---

### Stage 2: Instant Claim

**Touchpoint 2.1: Quick Setup**
```
[Claim Your DOG]
→ "Choose your username" (for receiving future sends)
→ @emma selected
→ Optional: Add email for notifications
```

**Touchpoint 2.2: Wallet Options**
```
"How would you like to store your DOG?"
→ [Simple: Keep in RuneBolt] ← Recommended
→ [Advanced: Send to my wallet]
```

**Touchpoint 2.3: Success**
```
"🎉 5,000 DOG claimed!"
→ Balance: 5,000 DOG
→ "You can now send to anyone with @username"
→ [Send DOG] [Learn More]
```

**Total Time:** 45 seconds
**Emotion:** Delighted → That was surprisingly easy

---

### Stage 3: First Experience

**Immediate Actions:**
- Explores the app
- May send some DOG back to friend
- Probably keeps most in RuneBolt

**Retention Factors:**
- Easy to use
- Can "cash out" to main wallet later
- Friends are on the platform

## Key Insights Across All Journeys

### Universal Friction Points
1. **Channel/liquidity concepts** — Must be hidden
2. **Address complexity** — Use usernames only
3. **Waiting for confirmations** — Set expectations clearly
4. **Recovery phrase anxiety** — Offer social recovery options

### Universal Delight Factors
1. **Speed** — "Instant" is magical
2. **Simplicity** — Feels like familiar apps
3. **Social** — Username search, memos
4. **Success moments** — Confetti, clear confirmations

### Conversion Optimization
- **90-second onboarding** — Critical threshold
- **First send within 2 minutes** — Retention correlates
- **Receive capability immediately** — Feel part of network
- **Social proof** — Show friends already using it
